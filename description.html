Data sets used

Sequence of numbers scaled to fit 256 x 256 window size. The sequence is garbled using random number generated by uniform distribution.

Algorithms simulated

The following algorithms are simulated at the same speed. 10 ms wait after each comparison operation.
  1. Bubble Sort - This algorithm has the worst performance on both data sets
  2. Selection Sort - This has the second worst performance.
  3. Insertion Sort - This performed well on half sorted inputs. Overall this is better than both bubble and selection sorts
  4. Quick Sort - This is the fastest of all. The average case was faster than all others (nlogn). The worst case is slower than merge sort
  5. Merge Sort - This is second best algorithm next to Quick sort. Average case is nlogn.
